# 云匠精品短剧创作引擎 — Drama Engine

## 项目简介

Drama Engine 是一个面向独立创作者和小型工作室的AI精品短剧全流程创作引擎。区别于市面99%的量产爽剧工具，本引擎专注于**现实题材、悲剧、家庭伦理、非遗文化**等需要文学功底和人物深度的短剧类型。

系统不替代创作者的审美判断和创作决策，而是将67000字创作方法论拆解为15个AI专家协作链路，帮助1人创作团队完成从"一个故事点子"到"一部完整精品短剧框架"的全流程。

核心理念：**创作者做决策，AI执行专业职能，文化给故事扎根。**

故事不是长在真空里的——红楼梦的盛大场面靠民俗文化撑着，鲁迅的祭祀场景靠乡土秩序驱动。引擎内置中华优秀传统文化知识库，让每一个场景、每一句对白都有文化根基，不再是精致的塑料花。

## 目标用户

- 独立编剧/创作者 — 有故事想法但缺专业反馈，反复试错成本高
- 非遗文化/现实题材创作者 — 现有AI工具只会生成套路化内容，无法处理人物灵魂和文学性
- 小型短剧工作室（1-3人）— 人力有限，需要AI承担多个专业岗位职能
- 影视专业学生 — 需要从点子到成稿的完整创作指导

## 核心痛点

当前AI编剧工具市场存在严重断层：

- 99%的工具教写量产爽剧：套路反转、霸总甜宠、爽感堆砌，同质化严重
- 精品短剧创作者无工具可用：现实题材、非遗文化等需要文学功底的类型，AI无法提供有效创作支持
- 1人创作团队的困境：独立编剧要同时承担选题、人设、结构、对白、视觉、合规等全部职能
- 人物扁平化：现有AI工具生成的人物是标签堆砌，没有灵魂、没有弧光、没有独特的语言方式
- 合规盲区：创作完才发现踩了审核红线，返工成本巨大

## 核心功能

**15个AI专家协作链路**（7个MVP已实现）：

| 专家 | 职能 | 对标人类角色 | 状态 |
|------|------|-------------|------|
| §0 灵魂捕手 | 对话式追问，确认故事方向 | 创意总监 | MVP |
| §1 角色铸造师 | 三层四维度人设 + 弧光追踪 + 语料库 | 人物设计师 | MVP |
| §2 合规守门员 | 创作前红线扫描 + 合规替代方案 | 法务审核 | MVP |
| §3 结构建筑师 | 15节拍表 + 23段落大纲 + 弧光追踪 | 编剧统筹 | MVP |
| §4 对白大师 | 个性化语料库 + 潜台词驱动 + 钩子链 | 对白编剧 | MVP |
| §8 项目配置师 | 将方向拆解为可执行的项目设定 | 制片人 | MVP |
| §13 视觉导演 | 光影系统 + 镜头系统 + 声音系统 | 视觉总监 | MVP |

**创作流程**：一个故事点子 → 7个AI专家接力 → 一部完整的精品短剧框架

## 差异化优势

| 维度 | 市面AI编剧工具 | 本引擎 |
|------|---------------|--------|
| 定位 | 量产爽剧生成器 | 精品短剧创作引擎 |

| 全球视野 | 无 | 7大传统18种叙事手法 |

## 技术架构

通用内核 + 项目配置双层设计，换项目只需替换§8配置：

- 核心引擎层：铁律系统（rules.py）+ 质量评分（scorer.py）+ 类型适配（type_adapter.py）
- 专家模块层：7个独立专家，可单独扩展
- 工作流编排层：§0→§2→§8→§1→§4→§3→§13 顺序编排
- 文化根基层：中华优秀传统文化知识库（民俗仪式+日常秩序+精神气质+文化意象），所有专家共享
- API接口层：CLI + FastAPI双入口

## 评测方向

- 点子→方向：§0灵魂捕手能否2-5轮对话内确认故事方向
- 方向→人设：§1角色铸造师能否生成三层四维度人物卡
- 合规预检：§2合规守门员能否识别红线并给出替代方案
- 全链路贯通：7个MVP专家顺序执行无断裂
- 通用性：仅替换§8配置即可适配新项目

## 项目边界

本引擎不是一键生成完整剧本的自动化工具——创作者必须做出关键决策；
不是量产爽剧套路生成器——专注需要文学功底和人物深度的类型；
不替代导演、制片等真人判断——AI执行专业职能，人类做审美决策。

它的定位是精品短剧创作全流程中的AI专业助手团队。
