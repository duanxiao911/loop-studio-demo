# §10 实战指挥（Mission Commander）

## 角色定位
端到端工作流编排专家。负责调度8+7个专家按正确顺序执行，处理异常恢复、断点续跑、进度监控。是整个引擎的"总调度台"。

## 核心能力
1. **流水线编排**：15个专家按依赖关系有序执行
2. **异常恢复**：单专家失败时自动重试（最多3次），3次失败跳过并标记
3. **断点续跑**：保存中间结果，意外中断后可从断点继续
4. **进度监控**：实时显示每个专家的执行状态

## 异常处理策略

| 异常类型 | 处理方式 |
|---------|---------|
| API超时 | 重试3次，间隔5s/15s/30s |
| 余额不足 | 暂停流水线，通知用户充值 |
| 输出格式错误 | 自动修复JSON，修复失败则重试 |
| 连续3次失败 | 跳过该专家，标记为"人工审核" |
| 用户中断 | 保存断点，下次可从断点恢复 |

## 断点续跑机制

1. 每个专家执行完成后，将输出序列化存入`checkpoint/`目录
2. 文件名格式：`{expert_id}_{timestamp}.json`
3. 恢复时扫描checkpoint目录，跳过已有结果的专家
4. 从第一个缺失结果的专家开始继续执行

## 输入
- `user_input`：用户故事方向
- `config`：引擎配置（API Key、模型、集数等）
- `resume_from`：断点恢复时指定从哪个专家继续
